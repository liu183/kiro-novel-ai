# 爽点菜谱

> 每个爽点一个文件，记录写法（铺垫 / 爆发 / 收尾 / 雷区）。
> 拆文中遇到该类型爽点时，把案例链接到对应菜谱。

## 菜谱结构

每个菜谱包含：
1. **核心定义**（一句话）
2. **触发条件**（什么场景适合）
3. **三段式写法**（铺垫 / 爆发 / 收尾）
4. **变体**（不同写法）
5. **雷区**
6. **案例引用**（链接到拆文）

---

## 新爽点的入库流程

1. 在拆文中识别新爽点 → 暂时记到 [`insights/`](../insights/)
2. 同样爽点在 ≥2 本拆文中出现 → 在此创建菜谱文件
3. 持续积累案例链接

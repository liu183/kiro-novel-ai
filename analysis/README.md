# 拆文分析层

> 经验原料库。所有原创创作经验来自这里的拆文积累 → 提炼 → 反哺到通用层和题材层。

---

## 📂 子目录

| 目录 | 用途 |
|---|---|
| `books/` | 按作品组织的拆文。每本一个文件夹 |
| `insights/` | 从多本拆文提炼出的"成品观察" |
| `beat-cookbook/` | 爽点菜谱（写法、铺垫、爆发、收尾） |
| `_template-拆文模板.md` | 拆文统一模板 |

---

## 🔄 拆文工作流

```
看完一本书
  ↓
[1] 复制 _template-拆文模板.md 到 books/<书名>/00-总览.md
  ↓
[2] 按章节细拆到 books/<书名>/章节拆解/
  ↓
[3] 提炼观察 → insights/<观察名>.md（标注来源）
  ↓
[4] 提炼爽点写法 → beat-cookbook/<层级>/<爽点>.md（标注来源）
  ↓
[5] 在 books/<书名>/00-总览.md 末尾标"已提炼至 ..."
```

---

## 📈 晋升机制

| 阶段 | 位置 | 触发条件 |
|---|---|---|
| 单点观察 | `books/<书>/00-总览.md` | 拆某书时偶然发现 |
| 凝练观察 | `insights/<观察名>.md` | 觉得这个观察可能在多本书复用 |
| 升级为通用技法 | `../universal/techniques/<技法名>.md` | 在 ≥3 本拆文中复现，证明跨题材普适 |
| 升级为题材套路 | `../genres/<题材>/tropes/<套路名>.md` | 仅在某题材内复现 |

晋升时：
- ✅ 保留 insights 原始版本（不删，作为出处）
- ✅ 在新位置加"出处"段落，链回 insights 和原始拆文
- ✅ 在 insights 文件顶部加 `> 已晋升至 [path]`

---

## 命名约定

- 作品目录：直接用作品名，如 `凡人修仙传/`
- 有版权顾虑可用代号：`某作品-A/`
- 拆文文件：`00-总览.md` / `人设拆解.md` / `节奏拆解.md` / `爽点拆解.md` / `暗线拆解.md`
- 章节细拆放 `章节拆解/chXXX-标题.md`
